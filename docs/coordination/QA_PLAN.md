## fixture_and_seed_catalog

### Notation and validity rules

- C(x,y) is a Cell. Snake arrays are head-first.
- GS(phase; snake; direction; queue; food; score; foodsEaten; tickMs; difficulty; endReason) lists every GameState field in contract order.
- External scheduler accumulator is always stated separately because it is not a GameState field.
- RNG[(upper -> return), ...] is the exact ordered ScriptedRandomSource expectation. Any missing or extra call fails immediately.
- Every executable fixture has integer in-bounds unique, Manhattan-contiguous snake cells; food is non-null and non-overlap outside menu/won; score equals foodsEaten times ten; reachable length equals foodsEaten plus three; queue length is at most two; tickMs follows the approved formula.
- P is the 400-cell serpentine path: for y=0..19, even rows enumerate x=0..19 and odd rows x=19..0. P[0]=C(0,0), P[397]=C(2,19), P[398]=C(1,19), P[399]=C(0,19).
- Seeded metamorphic checks may use seed 12345, but expected-value tests use ScriptedRandomSource so no RNG algorithm becomes product meaning.

## flakiness_and_negative_control_policy

- RNG/delta/command rule: every deterministic check names the exact FX fixture, command sequence, delta sequence, ordered RNG upper/return tuples, expected snapshot, and ordered events. Seed-only equality checks print seed and RNG implementation version but are not the sole oracle.
- Wait rule: sleep(), waitForTimeout(), and arbitrary timeout delays used as readiness signals are forbidden. Allowed waits are explicit DOM state, DomainEvent, MutationObserver, network completion, request failure, animation-frame condition, or bounded polling with a named condition and diagnostic deadline. Fixed windows that are themselves the oracle (250/400ms highlight, 5s no-auto-resume, 60s performance trace) use monotonic timestamps, animation frames, or recorded video and are not readiness sleeps.
- EV-FAIL-01 manifest fields, all mandatory: seed or scripted RNG tuple list; semantic command trace with dispositions; runtime/engine; exact runtime version; viewport or device/OS; full 40-character SHA; visual capture; execution trace/state JSON; console/stdout log; network/HAR log. Browser failures require actual screenshot/video and HAR. Node, static-source, and non-domain checks retain each field with a reasoned N/A plus the closest diagnostic substitute; they never silently omit a field.
- Flake gate: automated checks run three clean repeats at their matrix stage with retry disabled for the acceptance run. Any inconsistent result is a failure until reproduced with EV-FAIL-01; a later retry cannot convert it to PASS.
- Coverage: domain statement coverage>=90% and branch coverage>=90%; wall/self collision, food/non-overlap, reverse input, queue cap/consumption, unique-free, and full-board branches=100%.
