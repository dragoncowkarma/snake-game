## 수행 내용

- 변경 목적: 단일 Phaser `GameScene`과 얇은 application adapter를 추가했다. 순수 domain의 `reset`, `enqueueDirection`, `step`만 호출하고 점수·충돌·성장·속도 규칙은 Scene에서 다시 구현하지 않는다.
- 변경 파일:
  - `src/game/application-router.ts`, `fixed-step-scheduler.ts`: semantic command routing, 250ms delta clamp, frame당 최대 3 step, pre-step `stepDuration` capture와 pause/resume accumulator clear.
  - `src/game/game-scene.ts`, `board-renderer.ts`, `bootstrap.ts`: 480×480 / 20×20 Phaser Graphics board, FIT/center, single Scene mount와 snapshot/event rendering.
  - `src/game/input-controller.ts`, `lifecycle-controller.ts`: board-focus keyboard, same command route의 DOM button dispatch, hidden/blur/orientation pause, resize relayout과 cleanup.
  - `src/game/*test.ts`: scheduler duration/cap/clamp, READY Right, pause accumulator/queue, 20 restart의 subscriber baseline을 검증한다.
  - `src/main.ts`, `index.html`: browser에서 Phaser를 dynamic import하는 production entry.
  - `tsconfig.json`: TypeScript 6과 Phaser 4.2.1의 설치된 declaration 두 곳 호환성 오류만 건너뛰도록 `skipLibCheck: true`를 추가했다. project source 타입 검사는 유지한다.
- 의도적으로 변경하지 않은 범위: `src/domain/**`, `src/config/**`, `src/ui/**`, CSS, package/lockfile, tests/, accepted 계약·결정·상태 문서는 변경하지 않았다. `dist/`와 test artifact도 추적하지 않았다.

- 새 의존성: 없음. `package.json`과 lockfile 변경 없음.
