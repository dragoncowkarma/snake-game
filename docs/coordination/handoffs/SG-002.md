## 결정과 위험

- 새 의존성: spike 전용으로 `phaser@4.2.1`, `vite@8.1.4`, `typescript@6.0.3`, `@types/node@24.13.3`, `@playwright/test@1.48.2`를 추가했다. 제품 루트에는 추가하지 않았다.
- 새 결정 또는 기존 결정과의 차이: 제품·구현 결정은 없다. 검증 증거는 D-001의 Phaser 4.2.1 선택을 지지하지만 D-001은 H0b 전까지 `proposed`다. 계획상 Codex 5.6 Terra 대신 현재 제공된 GPT-5 Codex가 실행한 라우팅 편차가 있으며, 능력 동등성을 주장하지 않는다.
- TypeScript 제약: consumer 소스에는 `strict`, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`를 적용했다. Phaser 4.2.1 vendor declaration은 [자체 공식 TypeScript 생성 검증](https://github.com/phaserjs/phaser/blob/v4.2.1/scripts/tsgen/test/tsconfig.json)과 마찬가지로 `skipLibCheck: true`가 필요했다. vendor 파일을 patch하지 않았다.
- Playwright 제약: 최신 Playwright 1.61.1은 macOS 12용 WebKit을 설치하지 않는다. 1.52.0의 macOS 12 WebKit 2009 조합은 `FixedBackgroundsPaintRelativeToDocument` protocol mismatch로 시작에 실패했다. 공식 tagged browser/source 조합을 비교해 1.48.2를 이 runner에서 검증 가능한 조합으로 고정했고 두 브라우저가 통과했다. release 전에는 현재 지원되는 OS와 최신 브라우저 도구로 재검증해야 한다.
- visibility 제약: Playwright에 브라우저 공통 hidden-state 강제 API가 없어 `document.hidden` getter를 시험 중에 합성한 뒤 실제 `visibilitychange`를 dispatch했다. Phaser의 실제 HIDDEN/VISIBLE wiring을 관측하지만 OS 수준 tab backgrounding 자체를 자동화한 증거는 아니다.
- 번들 위험: Phaser 포함 단일 JS raw chunk가 1.38 MB라 Vite의 500 kB 경고가 난다. gzip은 359.17 kB이며 SG-002 실패 조건은 아니지만 SG-005/SG-009에서 product build budget과 code splitting을 다시 확인해야 한다.
- 저장소 상태: 작업 전부터 있던 `docs/DEVELOPMENT_PLAN.md`의 unstaged 사용자 변경은 건드리거나 staging하지 않았다.
- 남은 작업과 권장 담당자: Claude와 Antigravity가 task packet의 지정 설정으로 독립 리뷰하고 모델 라우팅 편차의 수락 또는 지정 모델 재실행 필요성을 판정한다. SG-001/SG-003/SG-004 증거가 모인 뒤 사람이 H0b에서 D-001~D-006을 승인하거나 수정한다.
