## 수행 내용

### 배경 조사에서 확인한 사실

- `src/game/input-controller.ts`(SG-012, Codex 소유)가 이미 board-focus 전용 키보드
  경로(capture-phase로 board 밖 키를 차단하고 board의 bubble listener만 명령을
  만든다)를 구현하고 있어, "이동/Pause/Mute 단축키는 focus가 보드에 있을 때만 직접
  처리한다"는 §3.4 요구사항은 이미 production에서 충족돼 있었다. 이 부분은 수정하지
  않았다.
- `src/ui/app.ts`의 Mute 버튼 `click` 리스너는 로컬 `muted` 변수를 직접 뒤집어 버튼
  텍스트/`aria-pressed`를 갱신했지만, keyboard `m` → `InputController` →
  `bootstrap.ts`의 공유 `dispatch` 경로는 이 로컬 변수를 전혀 건드리지 않았다. 이것이
  `DF-SG015-01`("키보드 `m` 음소거가 UI에 반영되지 않음")의 정확한 원인이었다.
- `preferences`(SG-016 `GamePreferences`, 난이도별 영속 최고 점수)는 `bootstrap.ts`에
  이미 존재했지만, `src/ui/app.ts`는 `let best = 0`으로 세션 로컬 메모리만 사용해
  화면에 그렸다 — 새로고침해도 영속된 최고 점수가 HUD에 반영되지 않는 별도의 미완성
  배선이었다.
- DEVELOPMENT_PLAN §3.5 "최고 점수 갱신은 게임 종료 화면에서만 강조한다"는 요구사항은
  어떤 코드에도 구현돼 있지 않았다.

### 변경 파일

- `src/ui/app.ts`: `GameShellHandle`에 `ShellMeta`(`muted`/`best`/`isNewBest`)와
  `updateMeta()`를 추가했다. `applySnapshot`은 이제 선택적 `metaPatch`를 받아 병합하고,
  `updateMeta`는 도메인 snapshot 없이(예: `toggleMute`처럼 `GameState`를 바꾸지 않는
  명령 뒤) 마지막 snapshot을 같은 meta로 다시 그린다. Mute 버튼 클릭 리스너는 더 이상
  로컬 `muted`를 직접 뒤집지 않고 `dispatch({ type: 'toggleMute' })`만 호출한다 —
  버튼 클릭과 키보드 `m` 모두 정확히 같은 `dispatch` 함수를 거치고, 화면 상태는
  오직 통합자가 호출하는 `updateMeta({ muted })`로만 갱신되어 두 경로가 항상 같은
  결과를 낸다.
- `src/ui/render.ts`: `renderSnapshot`의 meta 타입에 `isNewBest: boolean`을 추가했다.
  `ShellElements`에 `bestNote`(새 `<p class="hud__best-note">`, HUD 안, `result`와
  형제)를 추가하고 `createShell`에서 초기 `hidden = true`로 생성한다. 터미널
  (`gameOver`/`won`) 상태에서만 `bestNote.hidden = !meta.isNewBest`로 토글하며,
  기존 `result`(충돌/승리 사유 문장)의 `textContent`는 전혀 바꾸지 않는다 — 최초
  구현 시도에서는 `result` 문장 끝에 " New best!"를 이어붙였는데, 이는
  `tests/e2e/vertical-flow.spec.ts`(SG-014 소유, 이 packet의 허용 경로 밖)의
  `getByText('Game over (wall collision). Score 10.', { exact: true })` 정확 일치를
  깨뜨렸다(그 테스트의 실행 환경은 빈 localStorage로 시작해 첫 판의 점수 10이 항상
  "새 최고 기록"이 되므로). 별도 요소로 분리해 그 파일을 전혀 건드리지 않고 해결했다.
- `src/styles.css`: `.hud__best-note` 규칙을 추가했다 — 새 색상을 도입하지 않고
  이미 검증된 `--sg-text`/`--sg-surface-border` 토큰만 재사용해 굵기+테두리로
  강조한다(별도 대비 검증이 필요 없다).
- `src/game/bootstrap.ts`(narrow allowed-path 예외, 근거는 packet
  `allowed_paths_note` 참고): `createGameShell` 호출에 초기 `ShellMeta`(영속
  `preferences.snapshot.muted`, 현재 난이도의 `preferences.bestScore(...)`,
  `isNewBest: false`)를 전달한다. `dispatch` wrapper는 `toggleMute` 처리 뒤
  `shell.updateMeta({ muted: audio.isMuted })`를 호출한다(순환 참조를 피하려
  `shellRef.current` 헬퍼로 감쌌다 — `let shell` 단독 선언은 ESLint
  `prefer-const`에 걸린다). `application.subscribe` 콜백은 이제 `gameOver`/`won`
  전이일 때만 `preferences.recordScore(...)`를 호출하고 그 반환값(불리언)을
  `isNewBest`로 삼아 `shell.applySnapshot(state, events, { best, isNewBest })`에
  넘긴다 — 매 tick 호출하던 것에서 종료 시점 1회로 좁혔다(최종 점수는 항상 중간
  점수 이상이므로 영속되는 값은 동일하고, "새 기록" 신호만 정확해진다).
- `tests/e2e/qa-audit.spec.ts`(narrow allowed-path 예외): DF-SG015-01용
  `test.fail(true, ...)` 마커와 낡은 주석을 제거했다(버그가 고쳐졌으므로 이제
  실제로 통과해야 하고, `test.fail`이 걸린 채로 두면 통과 자체가 실패로
  보고된다). 같은 테스트 안에서 `muteButton` locator를 `getByRole('button', {
  name: 'Mute', exact: true })`에서 `page.locator('.mute')`로 바꿨다 — 토글 뒤
  버튼의 접근 가능한 이름이 "Mute"→"Unmute"로 바뀌므로 이름 기반 locator는 재평가
  시점에 더 이상 같은 버튼을 찾지 못했다(클래스는 안정적이라 안전).
- `docs/coordination/tasks/SG-017.yaml`, `docs/coordination/handoffs/SG-017.md`:
  이번 packet과 handoff.
- `docs/coordination/STATUS.md`: 이 handoff와 함께 갱신(아래 참고).
